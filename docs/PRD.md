## 2. 배경 & 문제정의

### 2.1 Problem Statement
런던/영국을 찾는 미술 애호가와 여행자는 **Tate / National Gallery / V&A / Royal Academy** 등 수십 개 기관의 전시 정보를 각자 웹사이트에서 따로 확인해야 한다. 한국어 블로그나 Time Out 같은 종합 매체는 일부만 다루고, 기간·가격·예매 링크가 흩어져 있어 **"이번 주 내가 볼만한 전시 3개"를 5분 안에 결정하기 어렵다.**

### 2.2 가설 (측정 가능)
- H1: 전시를 **한 목록 + 지도**로 묶어 보여주면, 사용자 세션당 탐색 전시 개수가 기존(기관별 서핑 대비) 3배 이상이 된다.
- H2: 공식 예매 페이지로 1-탭 이동이 가능하면, 전시 상세 → 예매 사이트 이동률(CTR) ≥ 8%가 된다.

### 2.3 기존 대안과의 차이
| 대안 | 약점 | GalleryUK의 차별점 |
|---|---|---|
| 각 미술관 공식 사이트 | 교차 비교 불가, 기관별 UX 상이 | 한 화면에 통합 |
| Time Out / Londonist | 광고 많고 예매 링크 일관성 낮음 | 광고 없음 + 예매 Deep Link 검증 |
| Google Maps 검색 | 전시 일정 정보 부실 | 기간/가격/카테고리 정제 |

## 5. 기능 명세 (MoSCoW)

### 5.1 Must Have — v1.0 (총 6개)

**F-01 전시 목록 (Home / List)**
- 현재 진행 중 / 곧 시작 탭
- 카드: 제목, 미술관, 기간, 썸네일
- 무한 스크롤 (또는 30개 단위 페이지네이션)
- 정렬: 최신 / 마감 임박 / 가까운 순 (위치 권한 시)

**F-02 전시 상세**
- 히어로 이미지, 제목, 미술관 링크, 기간, 가격(표기 또는 무료)
- 500~1000자 설명 (마크다운 렌더)
- 이미지 갤러리 최대 8장 (라이트박스)
- CTA: "예매하기" (티켓 URL 있을 때) + "지도에서 보기"

**F-03 미술관 지도**
- 런던 중심 기본 뷰, 마커 + 클러스터
- 마커 클릭 → Bottom Sheet(모바일) / Side Panel(데스크톱)에서 현재 전시 요약
- 현재 위치 토글 (권한 있을 때)

**F-04 키워드 검색**
- 상단 Search (sticky)
- 결과: 전시 / 미술관 섹션으로 분리
- 빈 상태, 로딩, 에러 UI

**F-05 예매 링크 이동**
- 외부 탭으로 공식 사이트
- `rel="noopener noreferrer"`
- `ticketUrl`이 null이면 "무료 / 티켓 불필요" 배지

**F-08 미술관 상세** (Gate 1 결정으로 Must 승격)
- 기본정보(이름·주소·운영시간·웹사이트)
- 위치 미니맵 + "Show on full map" 링크
- 현재 전시 / 예정 전시 그리드

### 5.2 Should Have

- F-06 필터 (카테고리 / 기간 / 지역 / 가격대) — URL 쿼리 반영
- F-07 근처 미술관 (Geolocation)

### 5.3 Could Have

- F-09 공유하기 (Web Share API)
- F-10 지도에서 "도보 N분"

### 5.4 Won't Have (v1.0)

- 로그인 / 계정
- 즐겨찾기 저장(서버), 알림
- 결제 / 티켓 발권
- 리뷰 / 평점
- 한국어 번역 UI (라벨은 영어, v1.2에서 i18n)

## 6. 비기능 요구사항 (NFR)

| 항목 | 기준 |
|---|---|
| 성능 (LCP) | 모바일 4G에서 ≤ 2.5s |
| Lighthouse (Perf/A11y/BP/SEO, 모바일) | 각 ≥ 90 |
| 접근성 | WCAG 2.1 AA |
| 브라우저 | Chrome/Safari/Firefox/Edge 최신 2 메이저 버전 |
| OS (PWA) | iOS 16+ Safari / Android 10+ Chrome |
| i18n | v1.0은 영어 (ko-KR은 v1.2) |
| 가용성 | Vercel 기본 (99.9%+ SLA) |
| 데이터 정확성 | 갤러리 기본정보 주 1회 검증 / 전시 종료일 지난 항목은 자동 hide |

## 7. 성공 지표 (KPI)

출시 후 8주 관측.

| KPI | 목표 |
|---|---|
| WAU | ≥ 500 |
| 세션당 평균 전시 조회 수 | ≥ 4 |
| 전시 상세 → 예매 사이트 CTR | ≥ 8% |
| 지도 인터랙션 비율 (세션 중 지도 진입) | ≥ 30% |
| PWA 설치 수 | ≥ 50 |
| Lighthouse 점수 4종 | 출시 시점 모두 ≥ 90 |
| 핵심 플로우 오류율 (세션 기준) | ≤ 1% |

## 9. 가정 / 의존성

- 초기 전시/미술관 데이터는 **수작업 시드 JSON** (`/public/data/*.json`)
- Google Maps API Key 발급 가능 (사용자 계정)
- Vercel 무료 티어로 충분 (대역폭 100GB/월 미만 예상)
- 이미지는 공식 사이트 hotlink 허용 범위 내에서 사용, 필요 시 대체 이미지로 교체
