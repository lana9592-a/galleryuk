### 🟡 Minor

**[m-01] 데이터 모델 스키마-예시 불일치 (이미지 URL 규칙)**
- 위치: `docs/data-model.md` §2 Gallery 예시 / §4 Zod 스키마
- 내용: `GallerySchema.logoUrl`이 `z.string().url()`로 선언됐으나, §2 표의 예시는 `"/images/galleries/tate-modern.svg"` (상대 경로). 실제 시드에서 이 값을 쓰면 Zod가 ValidationError를 던진다.
- 근거: Zod `string().url()`은 RFC 3986 절대 URL만 허용.
- 권고 방향(해결 지시 아님): 로컬 에셋을 쓸지 원격 URL만 쓸지 정책 결정 후, 스키마를 `z.string().url().or(z.string().startsWith('/'))` 같은 규칙으로 맞추거나 예시를 절대 URL로 수정.
- 영향: Phase 3 개발 시 빌드 실패. **Phase 2 진입 전까지 정책 결정 필요.**

**[m-02] 파생 상태(`status`, `daysLeft`) 갱신 타이밍 모호**
- 위치: `docs/data-model.md` §6 파생 필드 / `docs/user-stories.md` US-18
- 내용: US-18은 "빌드 시 오늘 기준 endDate < today인 항목 필터"라 서술했으나, `/` 및 `/exhibitions`는 SSG + ISR 1h로 합의됨(`docs/sitemap.md` §1). 빌드 시점 기준이면 ISR이 미발동한 구간에서 종료된 전시가 여전히 노출될 수 있다.
- 근거: Next.js ISR은 revalidate 간격마다 재생성하되, 사용자는 이전 스냅샷을 받을 수 있음.
- 권고 방향: (a) 파생 필드를 Server Component **렌더 시점**에 계산하도록 명시, 또는 (b) ISR 주기를 줄이고 날짜 경계마다 재생성되는 `revalidateTag` 패턴을 도입하도록 `agents/developer.md` Phase 4 체크리스트에 반영.
- 영향: 사용자 신뢰도(오래된 전시 노출). Phase 3 구현 전 결정 필요.

**[m-03] `openingHours` 스키마가 요일 키를 검증하지 않음**
- 위치: `docs/data-model.md` §4 Zod 스키마
- 내용: `z.record(z.string())`은 임의 키를 허용. 시드에 `"monday"` 또는 `"월"` 같이 섞여 들어와도 통과.
- 권고 방향: `z.object({ mon: z.string().optional(), tue: ..., sun: ... })`로 제한하거나 별도 enum 도입.
- 영향: 데이터 일관성 리스크 (Minor, Phase 4 API 연결 시까지 허용).

**[m-04] US-17 iOS 설치 유도 AC 불완전**
- 위치: `docs/user-stories.md` US-17
- 내용: "iOS Safari '홈 화면에 추가' 안내 도움말"만 기술돼 있고, **언제** 노출되는지(첫 방문 / N회 방문 / 공유 탭 누를 때) / **어디에** 노출되는지(모달 / 툴팁) / 디스미스 상태 보존 여부가 없음.
- 권고 방향: Designer가 Phase 2에서 트리거·위치·재노출 정책을 확정.
- 영향: 디자인 오너십으로 넘어갈 수 있으므로 본 Gate에서는 Minor로 유지.

**[m-05] MoSCoW와 Gallery 상세 스코프 충돌**
- 위치: `docs/PRD.md` §5 Must / `docs/sitemap.md` §1
- 내용: PRD는 "미술관 지도"(F-03)만 Must, "미술관 상세"(F-08)는 Should. 하지만 `/galleries/[slug]` 경로는 sitemap에 Should로 존재하고, US-07/US-08 플로우에 Sheet에서 "Open details" 링크가 등장. Should가 빠지면 Sheet의 링크가 Dead End.
- 권고 방향: (a) `/galleries/[slug]`를 Must로 격상, 또는 (b) Sheet의 "Open details" 대상 경로를 `/exhibitions?galleryId=` 같은 필터 목록으로 변경.
- 영향: v1.0 범위 명확화. Phase 2 디자인 시작 전 결정 필요.
