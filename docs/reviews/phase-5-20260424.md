## Findings

### 🔴 Blocker

### 🟡 Minor

- **[g5-01]** `NEXT_PUBLIC_SITE_URL` in Vercel is pre-set to `https://galleryuk.vercel.app` which happens to match the assigned alias; if the user later rewires a custom domain (`galleryuk.co.uk`, for instance), the env var must be updated in Vercel Settings → Environment Variables AND the deployment re-triggered for the canonical URL / OG image / sitemap to update. Keep this in mind when promoting to a vanity domain.
- **[g5-02]** Two of the 16 hero images in seed data are now Picsum placeholders (`siena-medieval`, `ra-summer-exhibition-2026`). When real-data ingestion happens in v1.1, replace with the gallery's official press imagery rather than Picsum.

### ⚪ Nit
- **[g5-n1]** SEO 92 (not 100) — Lighthouse's remaining deduction is likely the image-dimensions / `srcset` heuristic it applies differently to Next.js Image. Minor and hard to chase.
- **[g5-n2]** `pnpm test` is shipped locally but **not** wired into GitHub Actions CI. If v1.1 brings more contributors, set up a workflow so regressions can't land silently.

## Known Limitations Carried Into v1.0

1. **Seed data** — Not currently on the Definition-of-Done table as "real data". This is the primary limitation compared to the stated v1.0 DoD. All user-facing copy, URLs, and gallery descriptions are illustrative.
2. **Interactive map disabled** — `NEXT_PUBLIC_GMAPS_KEY` absent; fallback list is functional but not the richer experience implied by the PRD. Enabling is a single env-var + key restriction away.
3. **No custom domain** — https://galleryuk.vercel.app is the canonical URL until the user decides to attach a domain (backlog).
4. **No CI pipeline** — tests are hand-run; ensure that changes merged to `main` still `pnpm build` locally until CI is added.
5. **No authenticated / personalised surfaces** — favourites, notifications, recent views are all v1.1 scope per PRD.
